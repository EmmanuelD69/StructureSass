ETAPES DE BASE POUR MONTER UNE ARCHITECTURE SASS

  • 1 - créer un fichier package.json:
    LDC: npm init -y
  • 2 - installer les dépendances dont on a besoin:
    LDC: npm install --save(-dev) bootstrap / normalize / jquery / node-sass /...)
  • 3 - placer en ignore, 'node_module/' dans le .gitignore pour ne pas surcharger non repo git.
  • 4 - en cas ou on recupère un repo avec un 'package.json' deja configuré, faire un 'npm install' pour récuperer toutes les dépendances dont on a besoin.
  • 5 - indiquer dans le fichier 'package.json' la commande de script:
    "build-css": "node-sass style.scss style.css",
    commande qui va nous permettre de créer le fichier style.css via le fichier scss.
  • 6 - créer une structure de dossier correspondant à notre projet:
    ----> css
    ----> scss / main.scss
    Ps: pensez à créer le fichier 'main.scss'
  • 7 - mettre à jour dans 'package.json' la ligne "build-css" avec notre nouvelle structure de dossier aka:
    "build-css": "node-sass scss/main.scss css/style.css",
  • 8 - création du fichier index.htlm(ou php) et "linker" notre fichier style.css
    html:
  • 9 - mettre en place avec Sass l'option "watch" pour que le fichier scss soit mis à jour au fur et a mesure des changement. rajouter la commande --watch dans le fichier package.json sous la ligne script "build-css":
    "node-sass --watch scss/main.scss css/style.css",
  • 10 - installer "browser-sync" qui va nous permettre de recharger/raffraichir notre page de projet lorsqu'un ajout a été fait.
    LDC: npm install --save-dev browser-sync
  • 11 - créer dans le fichier package.json, le script qui va lancer browser-sync:
    "reload": "browser-sync start --server --files ."
    ATTENTION!! si messages d'erreur avec le '.', remplacer par une étoile(*)
  • 12 - lancer les scripts "build-css" & "reload" pour pouvoir commencer à travailler en temps réel:
    LDC: npm run reload &
    LDC: npm run build-css
  • 12bis - pour n'avoir qu'une fenetre de terminal LDC et lancer tous les scripts en meme temps, on peut installer la dependance (dev) "run-all" en
    LDC: npm install --save-dev npm-run-all
  • 13 - créer le script "watcher" dans package.json pour lancer toutes les scripts en même temps en 1 ligne de commande:
    "watcher": "run-p build-css reload"
  • 14 - lancer la commande LDC qui va démarrer notre script "watcher":
    LDC: npm run watcher
    ATTENTION!!! Si votre fichier "index" est en PHP, il y'aura un message d'erreur car on demande à "watcher" de détourner le fichier php gérer par le serveur apache de notre machine et de le gerer virtuellement, il n'y arrivera pas! laisser le fichier index en HTML (aka: index.html)
  • 15 - mise en place d'une architecture Sass avec un dossier "base" pour intégrer nos fichiers "partials".
    on créer un dossier 'base' sous le dossier 'scss'
  • 16 - on créer un fichier '_reset.scss' dans le dossier 'base'
  • 17 - dans ce fichier '_reset.scss' on va copier/coller le model de "Eric Meyer" situé à l'adresse suivante:
    https://github.com/O-clock-Cosmos/Modele-projet-front/blob/master/css/reset.css
  • 18 - Repasser sur le fichier 'main.scss' pour commencer à "include" les différents éléments (fichiers) qui vont nous permettre de recoller tout notre css en un seul block
  • 19 - Créer le sous dossier 'abstract' situé dans le dossier 'scss'
  • 20 - Créer le fichier '_variables.scss' dans le sous dossier 'abstract'
  • 21 - Importer le fichier '_variables.scss' dans notre fichier 'main.scss'
  • 22 - Créer le sous dossier 'components' dans le dossier 'scss'
  • 23 - Créer le fichier '_buttons.scss' dans le dossier 'components'
  • 24 - Importer le fichier '_buttons.scss' dans notre fichier 'main.scss'
  • 25 - Créer tous les sous dossier et fichiers scss dont vous avez besoin et les inclurent dans le fichier 'main.scss' en respectant bien l'ordre logique:
    ---->Import des utilitaires.
    @import 'abstract/variables';
    ---->Import des éléments de base
    @import 'base/reset';
    @import 'base/typo';
    ---->Import des composants
    @import 'components/boutons'
    @import 'components/articles'
  • 26 - Oui je sais, mon CSS est moche, mais ne pas tenir compte de cette partie pour votre projet :p , amusez vous bien!
